
## 1. Descripción
Los documentos de planificación constituyen el conjunto de lineamientos que orientan el desarrollo del proyecto dentro del repositorio del equipo, permitiendo organizar tareas, recursos y tiempos de manera estructurada. Estos documentos facilitan el seguimiento del progreso mediante herramientas colaborativas como GitHub, asegurando el cumplimiento de los objetivos definidos.

## 2. Plan del proyecto
Incluye:
- Definición de roles del equipo 
- Organización del repositorio (/docs, /src) 
- Uso de ramas (main, develop, feature) 
- Gestión de tareas con issues 
Propósito: garantizar una ejecución ordenada y colaborativa del proyecto.

## 3. Plan de alcance
Define las funcionalidades que serán desarrolladas en el sistema, tomando como referencia los artefactos documentados en el repositorio.
Incluye:
- Módulos principales del sistema 
- Historias de usuario (Product Backlog) 
- Entregables documentados en /docs 
Propósito: mantener el enfoque del proyecto y evitar desviaciones innecesarias.

## 4. Plan de tiempo
Organiza el desarrollo del proyecto en sprints, los cuales se reflejan en la planificación del backlog y en el avance registrado en el repositorio.
Incluye:
- División del trabajo por iteraciones 
- Seguimiento del progreso mediante commits 
- Entregas parciales por sprint 
Propósito: asegurar el cumplimiento de los tiempos establecidos.

## 5. Plan de riesgos
- Identifica posibles dificultades relacionadas tanto con el desarrollo técnico como con el trabajo colaborativo en GitHub.
- Riesgo	Probabilidad	Impacto	Mitigación
- Conflictos en el repositorio	Media	Media	Uso de ramas y pull requests
- Pérdida de información	Baja	Alta	Control de versiones en Git
- Retrasos en tareas	Media	Media	Seguimiento por issues
- Problemas técnicos	Media	Media	Pruebas tempranas
Propósito: reducir riesgos asociados al desarrollo y colaboración.

## 6. Plan de calidad
Define los criterios que debe cumplir el sistema y la documentación almacenada en el repositorio.
Incluye:
- Buenas prácticas de codificación 
- Documentación clara en /docs 
- Validación de funcionalidades 
- Revisión de código mediante pull requests 
Propósito: asegurar un producto funcional y bien documentado.

## 7. Plan de comunicación
Establece los mecanismos de interacción entre los integrantes del equipo utilizando herramientas digitales.
Medios:
- GitHub (issues, comentarios, pull requests) 
- Herramientas de mensajería (WhatsApp, Discord) 
- Reuniones de seguimiento 
Propósito: mantener una comunicación continua y efectiva.

## 8. Plan de recursos
Define los recursos necesarios para el desarrollo del proyecto.
Recursos humanos:
- Integrantes del equipo con roles asignados 
Recursos tecnológicos:
- GitHub (repositorio y control de versiones) 
- Lenguajes y frameworks definidos 
- Herramientas de desarrollo 
Propósito: asegurar el uso eficiente de los recursos disponibles.

## 9. Plan de pruebas
Define cómo se validará el sistema desarrollado dentro del repositorio.
Incluye:
- Pruebas funcionales 
- Validación de módulos implementados 
- Revisión de código en pull requests 
Propósito: detectar errores y garantizar el correcto funcionamiento del sistema.

## 10. Consideraciones
- El desarrollo se gestiona de forma colaborativa mediante GitHub 
- Se prioriza un enfoque incremental basado en sprints 
- Todos los avances deben registrarse mediante commits 
- La documentación debe mantenerse actualizada en el repositorio
